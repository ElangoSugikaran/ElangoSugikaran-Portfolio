DentAssist-LK

AI-powered Dental Appointment & Management System

View Source Code on GitHub Visit Live Site

DentAssist-LK is a full-stack web application built to simplify online dental appointment booking. The system allows users to create an account, log in securely, and book appointments through a guided multi-step process. It checks real-time doctor availability, lets users select date, time, and appointment type, and sends an automated confirmation email after the booking is completed using Resend. The application includes role-based dashboards for both users and admins. Users can view their upcoming appointments, while admins can manage doctors, view all appointments, and update appointment statuses. DentAssist-LK is built with Next.js (App Router), Prisma ORM, and Neon PostgreSQL, with secure authentication and authorization handled by Clerk. It also integrates a real-time AI voice assistant powered by VAPI, allowing users to receive quick dental information and interact more naturally with the system. This project was created to upskill and explore modern full-stack development using clean architecture, reusable component design, and best practices for authentication, database modeling, and state management.

Project Details


Project Title DentAssist-LK — AI-powered Dental Appointment Platform
Project Overview

DentAssist-LK is a full-stack Next.js application that enables patients to find dentists, book appointments, and receive confirmations, while providing administrators tools to manage doctors, view appointments, and monitor system analytics.

Core features:

  • Secure authentication with Clerk and automatic user sync via webhooks.
  • Multi-step appointment booking with date/time availability checks and appointment types (checkup, cleaning, emergency).
  • Admin dashboard for doctor management, appointment oversight, and analytics.
  • AI voice assistant integration (Vapi.ai) for conversational dental guidance (Pro feature).
  • Email notifications using Resend and templated emails built with React Email.
Skills & Competencies Demonstrated
  • Server & client components in Next.js (App Router).
  • TypeScript, state management with Zustand and TanStack Query.
  • Prisma ORM + Neon PostgreSQL schema design and migrations.
  • Authentication & role-based authorization with Clerk and webhook handling.
  • Form validation using React Hook Form + Zod.
  • Real-time voice integration and production-ready email workflows.
  • Accessibility and reusable UI with Radix + shadcn/ui and Tailwind CSS.
Technology Stack
  • Frontend: Next.js (App Router), React 19, Tailwind CSS, shadcn/ui, Radix
  • State & Data: Zustand (client state), TanStack Query (server state)
  • Backend / DB: Prisma ORM, Neon PostgreSQL
  • Auth: Clerk (webhooks for user sync)
  • AI & Voice: Vapi.ai integration
  • Email: Resend + React Email templates
  • Extras: Recharts for analytics, date-fns for date handling
  • Deployment: Vercel (recommended), Database on Neon/Supabase/Railway
Notable Implementation Patterns
  • Server Actions for secure server-side operations (booking, doctor CRUD).
  • Webhooks (Clerk) to keep the application database in sync with the auth provider.
  • Type-safe Prisma schema and migrations for reliable DB changes.
  • Zod schemas + React Hook Form for validated, type-safe forms.
Deployment & Env

Important env vars:

DATABASE_URL=postgresql://...
CLERK_SECRET_KEY=...
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=...
RESEND_EMAIL_API_KEY=...
VAPI_API_KEY=...

Recommended hosting: Vercel for the Next.js app and a hosted PostgreSQL provider (Neon / Supabase / Railway).
